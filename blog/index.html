Arrays VS Hashes
An array are: ordered, integer-indexed collections of any object.

Lets break this down a little bit... So what this means this that an array is a collection or "list" of an object(string,integer,etc...) these objects are ordered using and integer-indexing, starting at 0. So the first value will be indexed at 0 the second 1, third 2 etc...

array_name = [0, "one", 2, "three"]

Hashes

Hashes are: a collection of unique keys and their values.

These are really similar to arrays, but lets first break down what the line above means. So a hash is a collection or "list" of keys, now these keys are very similar to the integer-index that arrays use, but for keys we can define them as anything, such as a string or integer. Then the values can be any object just like in arrays.

CSS: inline VS inline-block.

I'm going to try and make this as basic as I can for everyone, this topic is a bit new for me but after some searching around here's what I've found:

inline - Inline takes the left and right margins and padding into consideration but not the top or the bottom. This also allows elements to sit on any side of them. This means that elements will be stacked on top of each other without spacing but can have space from right to left.

inline-block - Inline-block takes all margins and padding into consideration; top,right,bottom,left. This allows for the elements to be in a row with margins or padding.

Git Hub and Version Control

A Very Basic Overview

The command line, git and GitHub are the staples of coding and version control. GitHub is a repository hosting service, meaning it can hold all of our files and folders when coding. It’s great interface also makes it easy for managing and sharing projects/repos a.k.a. repositories. The command line matched with git is how we communicate to GitHub.

Using CLI (command line) we can “git clone” or copy our repos to our computer. Doing this makes sure that we can edit our files offline, we can also get to them from anywhere, by cloning/pulling them to any computer. When you make an edit on the file locally, it does not automatically update online, you must tell it to do so, this helps prevent from sharing broken code.
